The Admin section gives workspace administrators powerful tools for monitoring, managing, and maintaining your workspace. Access live activity feeds, remote admin capabilities, and comprehensive audit logging.

## Admin Dashboard

Navigate to **Admin** from the sidebar to access the admin dashboard. The dashboard provides a real-time overview of your workspace activity.

<Frame>
  <img src="/images/admin-dashboard.png" alt="Admin dashboard overview" />
</Frame>

### Live Activity Feed

The activity feed shows real-time actions happening across your workspace:
- Members joining or leaving games
- Promotions, demotions, and strikes being issued
- Applications submitted
- Articles created or updated
- Approval actions taken

The feed updates automatically so you always have a current picture of what's happening.

## Remote Admin

Remote admin capabilities let you take action on your Roblox game servers directly from the dashboard.

### Server Inspection

View the status of your tracked game servers:
- Active player count
- Server uptime
- Connected members and their roles

<Frame>
  <img src="/images/admin-servers.png" alt="Server inspection view" />
</Frame>

### Remote Actions

Depending on your game tracking setup, you may be able to:
- View player lists across servers
- Monitor active sessions and events
- Check server health and connectivity

## User Management

From the Admin section, manage your workspace members:

<Steps>
  <Step title="Search for a member">
    Use the search bar to find members by name or Roblox username.
  </Step>
  <Step title="View member details">
    See their full profile including activity, history, permissions, and current status.
  </Step>
  <Step title="Take action">
    Perform administrative actions:
    - Update permission group assignments
    - Issue strikes, demotions, or suspensions
    - Assign courses
    - Review their complete logbook history
  </Step>
</Steps>

## Bulk Actions

For managing multiple members at once:

1. Select members from the member list
2. Choose a bulk action (e.g., assign to permission group, assign course)
3. Confirm the action

<Warning>
  Bulk actions affect multiple members at once. Double-check your selection before confirming.
</Warning>

## Audit Logs

The audit log records all administrative actions taken in your workspace. This is separate from the [Logbook](/using-roteam/logbook) and focuses specifically on admin-level changes:

- Permission changes
- Settings modifications
- User management actions
- Workspace configuration updates

<Frame>
  <img src="/images/admin-audit.png" alt="Audit log" />
</Frame>

### Filtering Audit Logs

Filter audit logs by:
- **Action type** — What kind of action was taken
- **User** — Who performed the action
- **Date range** — When the action occurred

## Workspace Settings

Admins can access all workspace settings from **Settings** in the sidebar. Key admin settings include:

- **Workspace Info** — Name, emblem, and general configuration
- **Members & Roles** — Permission groups and tracked roles
- **API Keys** — Integration credentials for game tracking and auto ranking
- **Billing** — Subscription management
- **Danger Zone** — Delete workspace (owner only)

<Note>
  Only users with admin-level permissions can access the Admin section. The workspace owner (rank 255) always has full admin access.
</Note>

<Tip>
  Check the admin dashboard regularly to stay on top of workspace activity and catch any issues early.
</Tip>
